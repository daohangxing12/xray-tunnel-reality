# xray-tunnel-reality

Interactive one-command Xray installer with three modes:

1. VLESS Reality Vision using a 3x-ui style tunnel structure:

```text
public port -> tunnel inbound -> 127.0.0.1 local VLESS Reality Vision inbound
```

2. SOCKS5 inbound with username/password authentication.

3. Cloudflare preferred IP/domain VLESS + WebSocket:

```text
client -> CF preferred IP/domain:443 -> Cloudflare proxied domain -> origin VLESS-WS port
```

## Interactive install

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/daohangxing12/xray-tunnel-reality/main/install.sh)
```

The script asks you to choose:

```text
1) VLESS Reality Vision
2) SOCKS5
3) CF优选 VLESS-WS
```

## Non-interactive examples

Reality with defaults:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/daohangxing12/xray-tunnel-reality/main/install.sh) --mode reality --yes
```

Reality with custom port and SNI:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/daohangxing12/xray-tunnel-reality/main/install.sh) --mode reality --port 56778 --inner-port 4432 --sni www.tesla.com
```

SOCKS5:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/daohangxing12/xray-tunnel-reality/main/install.sh) --mode socks5 --port 21109 --user nt --pass nt888888
```

Cloudflare preferred IP/domain VLESS-WS:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/daohangxing12/xray-tunnel-reality/main/install.sh) \
  --mode cf-ws \
  --port 31520 \
  --cf-domain hostdzire.212202.xyz \
  --cf-entry cf.3666888.xyz \
  --path /ws233
```

`--cf-entry` is the client address. It can be a preferred IP, preferred domain, or your own domain pointed to a preferred Cloudflare IP, for example values selected from a Cloudflare preferred-IP site.

`--cf-domain` is the Cloudflare proxied/orange-cloud domain used as WebSocket Host and SNI.

## Defaults

Reality mode:

- Public tunnel port: `56777`
- Local Reality port: `4431`
- SNI: `www.tesla.com` unless `--sni` is provided
- UUID: random
- Short ID: random

SOCKS5 mode:

- Public port: `21109`
- Username/password: prompted or random with `--yes`
- UDP: `false`

CF VLESS-WS mode:

- Origin/source port: `31520`
- WebSocket path: `/ws233`
- UUID: random
- Cloudflare domain: prompted or `--cf-domain`
- Preferred entry: prompted or `--cf-entry`; defaults to the Cloudflare domain

Common:

- Service: `xray-tunnel-reality`
- Config: `/etc/xray-tunnel-reality/config.json`

## Cloudflare setup for cf-ws

DNS:

```text
cf-domain.example.com  A  origin-server-ip  Proxied/橙云
```

Origin Rule:

```text
If incoming request matches:
  http.host eq "cf-domain.example.com"

Then:
  Rewrite destination/origin port to your source port, for example 31520
```

Client URL generated by the script:

```text
vless://UUID@CF_ENTRY:443?encryption=none&security=tls&type=ws&host=CF_DOMAIN&sni=CF_DOMAIN&path=%2Fws233#NAME
```

## Relay Panel

For Reality/SOCKS5, use the printed target:

```text
landing-ip:port
```

For CF VLESS-WS, the client connects through Cloudflare, so configure Cloudflare DNS and Origin Rule instead of a normal relay target.

## Manage

```bash
systemctl status xray-tunnel-reality --no-pager
systemctl restart xray-tunnel-reality
journalctl -u xray-tunnel-reality -f
```

## Notes

- Do not commit passwords, SSH keys, panel secrets, or GitHub tokens.
- Re-running the script overwrites `/etc/xray-tunnel-reality/config.json`.
- If a port is already in use, stop the old service or use another `--port`.
- CF VLESS-WS origin is plain WS; TLS is provided at Cloudflare edge. Match your Cloudflare SSL mode and Origin Rule accordingly.
